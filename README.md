# 🍣 寿司餐厅 - Sushi Project

一个用 Python 和 Pygame 开发的趣味经营类游戏，玩家需要在有限的时间内为顾客制作寿司和饮品，赚取小费来通过关卡。

## 📋 项目概述

这是一个简单而有趣的时间管理游戏，灵感来自于经营类游戏。玩家在虚拟寿司餐厅中：
- 从食材容器中选取米饭和配料
- 在菜板上制作寿司
- 制作饮品（清酒、啤酒、味增汤）
- 为等待的顾客提供订单的寿司和饮品
- 通过赚取顾客的小费来通过关卡

## 🎮 游戏特性

### 核心玩法
- **食材系统**：米饭、章鱼、扇贝、三文鱼、金枪鱼
- **饮品系统**：清酒、啤酒、味增汤
- **顾客管理**：同时最多 3 个顾客排队等待
- **计时系统**：每场游戏 60 秒，每个订单 10 秒

### 进度系统
- **关卡系统**：游戏保存进度，支持无限关卡
- **目标金额**：
  - 第 1 关：100 元小费
  - 每关增加：50 元
- **小费计算**：
  - 订单完全正确：20 元
  - 订单对一半：10 元
  - 订单错误：0 元

### 视觉和音频
- 背景音乐和音效系统
- 顾客动画（等待、开心、生气）
- 游戏结束屏幕（胜利/失败）
- HUD 显示：计时器、小费统计、关卡数

## 🎯 游戏规则

### 基本操作
- **左键点击** 食材容器 → 添加到菜板
- **点击菜板** 完整寿司 → 拿起寿司
- **左键点击** 饮品机 → 获取饮品
- **左键点击** 顾客区域 → 送出手中物品

### 制作寿司流程
1. 点击米饭容器 → 添加米饭到菜板
2. 点击配料容器 → 在米饭上添加配料
3. 寿司完成后，点击菜板拿起
4. 点击等待的顾客将寿司交付

### 游戏进度
- 时间结束后，系统计算小费
- 小费 ≥ 目标金额 → 通过关卡，升级
- 小费 < 目标金额 → 关卡重试

```python

FPS = 60

# 游戏时长

# 关卡系统
INITIAL_TARGET_TIPS = 100      # 第一关目标
TARGET_TIPS_INCREMENT = 50      # 每关增加量

# 订单时限
ORDER_DURATION_SECONDS = 10
```

## 🎨 游戏元素

### 容器和工作台
- 米饭容器、配料容器（章鱼、扇贝、三文鱼、金枪鱼）
- 菜板（用于组装寿司）
- 饮品机（清酒、啤酒、味增汤）

### 顾客系统
- 顾客有三种状态：等待、开心、生气
- 每个顾客有随机生成的订单
- 订单包含寿司类型和饮品类型

### UI 元素
- 全局计时器（倒计时）
- 小费显示（当前/目标）
- 当前关卡显示
- 订单计时器（显示订单截止时间）

## 📝 主要类和函数

### config.py

## 🎼 游戏状态

游戏有三种主要状态：

1. **START_SCREEN** - 开始屏幕
   - 显示当前关卡
   - "开始游戏"按钮
   - "重置进度"按钮

2. **GAME_RUNNING** - 游戏进行中
   - 显示食材、菜板、顾客
   - 实时计时器和小费统计
   - 处理玩家输入

3. **GAME_OVER** - 游戏结束
   - 显示"时间到"画面
   - 显示胜利/失败结果
   - 提示是否继续或重试

## 💾 存档系统

重新启动游戏时自动加载，支持重置按钮恢复到第 1 关。
